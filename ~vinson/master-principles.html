1. Principle: Don't Waste Space by ~eduard

Example

The components tab on the Jetstrap website has too much wasted space. This will make finding certain components difficult.

Explaination/Suggestion

Explination:

Space is an integral part of a webpage. It provides structure and can improve the readability of any webpage. A designer must conscious of how a webpage incorporates space into its design. However, sometimes a webpage can contain elements with too much space between them. This can lead to a webpage that lacks content or structure which can engatively affect the user's experience.


Suggestion(s):

  • Place the name of the component next to the "drawing" so that the space is nor wasted.
  • Create a two-column layout so that two components are displayed at each line.

2. Principle: Don't Put Me In Chains by ~eduard

Example

The windows for the Jetstrap editor cannot be adjusted by the user which results in some wasted space in the "Screens" window.

Explaination/Suggestion

Explination:

Giving the user the ability to customize the layout of a website (such as the window panes of an editor) can greatly improve the user experience. The importance of an element will depend on the user so it maybe necessary to give the user the ability to minimize/remove the elements that they do not need to maximize the screenspace for the elements that do need. A layout will not appeal to all users so some customization options can improve the user's experience.


Suggestion(s):

  • Allow window panes in the editor to be adjusted by the user.

3. Principle: Order Is Important by ~eduard

Example

The ordering for the elements in the Component window does not place the more important items at the top of the list. The elements do not follow a noticable pattern for the user.

Explaination/Suggestion

Explination:

The order of elements is something that should be taken into consideration when designing a website. Generally, the most important items should come first to decrease the search time for the user and increase the website's usability. The ordering for elements should be based around the needs of the user.


Suggestion(s):

  • Order the components window such that the most often used elements are placed at the top.
  • Group related elements (e.g. all the form input elements) next to each other.

4. Principle: Ignore Standards If You Can Do Better by ~eduard

Example

The project listing is sorted by date rather than importance. The new project button is always the first element.

Explaination/Suggestion

Explination:

In general, standards are something that a web designer should follow. However, in some cases, it may be better to ignore the standards. This can be things such as ordering a list alphabetically instead of by importance or using a different color to highlight important parts of the website.


Suggestion(s):

  • If the order of the component window is not by importance, perhaps alphabetically would make it easier to find a specific component.
  • Give the user the ability to sort lists by different metrics (i.e. importance, date, alphabetically etc;).

5. Principle: Make Effective Use Of Whitespace by ~eduard

Example

The editor page is a bit cluttered due to lack of whitespace between elements. However, this only applies if all windows are opened.

Explaination/Suggestion

Explination:

Whitespace is a very important part of designing a website. Effective use of whitespace can help improve the usability for the users and help convey information easier. Giving elements room to breathe allows the elements to standout from the page which reduces the users cognitive load which leads the readability of the user. However, one must be conscious of having too much whitespace since that may violate the principle of "Don't Waste My Space".


Suggestion(s):

  • Stack the windows on a single side and allow them to be minimized to increase the screenspace for the view and code windows. This should help increase the space for those elements.
