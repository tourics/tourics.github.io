Principle #1

Don't waste my space

Description:

Navigation bar has a lot of emtpy space on top

Suggestion:

Put the frenqencely used components icon on it, so users can quickly access the component without opening the left pane, which give more space to the design view screen and used up the empty space on the navigation bar.

Principle #2

Don't put me in chains

Description:

The positions of all the components that dragged into the home design screen are nearly fixed and not very flexible

Suggestion:

The tool should be redesigned to enable the user to modefied the component position in the screen, For example, if I want the title of the paragrah to be in the center a little bit, I could just drag it to the right.

Principle #3

Order is importan

Description:

The property pane at the right side is auto displayed when user opening up this web tool, which is empty because there is not thing selected in the screen when user new to this tool

Suggestion:

The property pane should be minimized because it is not useful to new user who first launch to this tool. It should be pop up right after user click on a component in the design screen.

Principle #4

Ignore standards if you can do better

Description:

Some standards component of bootstrap sucks, E.g the radio buttons group are all toggleable which means can't do single selection.

Suggestion:

They should implement a function that we can add our own code into their component elemnts for advance user

Principle #5

COLOURS is important

Description:

The color of the round shape "test or build" switcher on top is almost the same color as the navigation bar. User will have a hard time figure out there is a button up there or not

Suggestion:

It should use a brighter color for that switcher button like oragin or blue.

Principle #6

Don't waste my time

Description:

the delete button of the component is too small to find and it takes 2 step to finish the delete action(click on the component and click the trash can icon, or right click the component and click on the delete).

Suggestion:

They should implement drag to delete function so user could drag the element out of the browser to delete the component in the home design screen.
